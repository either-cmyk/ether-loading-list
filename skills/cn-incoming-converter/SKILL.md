# CN 출고내역 → 에이투지 간편입고 변환 스킬

## 개요

CN인사이더에서 받은 출고내역 엑셀(S-QED / S-QDE)을 사방넷 풀필먼트(에이투지) **간편입고 등록**
양식으로 변환한다. 상품코드(B열)는 스킬에 내장된 `product_codes.json`(라벨명→E코드, 597건)으로
자동 매핑하므로 **별도의 공산품 상품코드 파일 업로드가 필요 없다.**

## 입력

- **출고내역 엑셀 1개** (필수) — 파일명에 `S-QED` / `S-QDE` 또는 `출고내역` 포함
  - 1행: 타이틀 (`박스수량:N(N상자+0PL)` 포함)
  - 2행: 헤더 (B=상자번호, F=SKU/바코드, H=라벨명, K=출고수량, L=세트수량)
  - 3행~: 데이터, 마지막 행: TOTAL

## 변환 규칙

### 수량 결정 (행 단위)
- L열(세트수량)이 있고 0보다 크면 → **L열** 사용
- L열이 비었거나 0이면 → **K열(출고수량)** 사용
- (조합세트 행은 보통 L이 비어 K를 세트수로 사용)

### 상품 그룹핑
- 바코드(F열 SKU) 기준으로 동일 상품을 한 행으로 합산
- 수량은 전체 합계

### 박스번호 표기
- 연속 박스는 `~`로 범위: `S-QED-N-001~S-QED-N-003`
- 불연속 박스는 `,`로 구분: `S-QED-N-003, S-QED-N-005`
- 혼적(여러 SKU가 한 박스) 시 박스번호가 SKU마다 콤마로 분리되어 표기됨

### 정렬
- 라벨명(H열) 오름차순

### 상품코드 매핑
- 출고내역 라벨명(H열)을 **정규화**(모든 공백 제거 + 끝 콤마 제거)하여 내장 코드표에서 E코드 조회
- **바코드(출력 E열)** 는 출고내역 SKU(F열)를 그대로 사용 → 항상 정확
- 매칭 안 되는 라벨명은 상품코드 빈칸 + 경고 출력 (수동 확인)

## 출력 양식 (사방넷 간편입고 등록, 시트명 `엑셀입고`)

| 열 | 헤더 | 내용 |
|---|---|---|
| A | 박스넘버 | 박스번호 범위 |
| B | 출고상품코드 | E코드 (내장 매핑) |
| C | 상품명 | 라벨명 |
| D | 수량 | 합산 수량 |
| E | 바코드 | 출고내역 SKU |
| F | 유통기한 | 비움 |
| G | 로케이션 | 비움 |
| H | 입고메모 | 비움 |

폰트: 맑은 고딕 10pt, 헤더 볼드, 전체 thin 테두리.

## 실행

```bash
python skills/cn-incoming-converter/scripts/convert.py <출고내역.xlsx> [출력.xlsx]
```

- 출력 파일명 기본값: `{코드}_{사업자명}_간편입고등록.xlsx` (예: `S-QED260618_이더컴퍼니_간편입고등록.xlsx`)
- 상품코드표를 일시적으로 다른 파일로 덮어쓰려면: `--codes <매핑.json | 공산품.xlsx>`
- 완성 파일은 **사용자가 선택한 마운트 폴더**에 저장 (outputs 임시폴더 금지)

## 상품코드표 갱신

`scripts/product_codes.json`은 사방넷 이더컴퍼니 계정 공산품 E코드(라벨명→코드) 스냅샷이다.
신규 상품이 추가되어 매칭 실패가 나면, 최신 재고조회를 기준으로 이 파일을 갱신하고 플러그인
버전을 bump 후 push한다.

## 주의사항

- TOTAL 행 제외, 머지셀 안전 처리(`data_only=True`)
- 검증: 그룹 합계 == 행 합계 (내부 일관성). 원본 TOTAL의 L열은 조합세트 행을 빼고 집계되므로
  단순 비교 시 어긋날 수 있음 — 행 합계 기준이 정답
- 동일 라벨명이 두 E코드에 매핑되는 소수 케이스(아쿠아슈즈 블랙 230-235 등)는 바코드로만
  구분 가능 → 매칭 실패 시 경고 확인
