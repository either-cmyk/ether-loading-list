# 쿠팡 팔레트 적재리스트 생성 스킬 (v1.1.0)

## 개요

거래명세서 PDF + CN인사이더 출고내역 엑셀(S-QED) 두 파일을 받아 **팔레트별 1페이지**로 구성된 적재리스트 XLSX + PDF를 동시 생성한다.

**v1.1.0 변경점 (2026-05 학습 반영):**
- N-G 트레이 (가습기 시리즈) 지원
- 혼합 트레이 (한 트레이에 G/N-G 박스 동시) 지원
- "이미지" 추가 컬럼 변형 자동 감지 (열 위치 헤더 이름 기반 조회)
- 18+ 행일 때 PDF 행 높이 자동 축소 (A4 1페이지 fit)
- 자동 혼적 감지 우선 + 사용자 명시 화이트리스트 합집합

**v1.0.0 (2026-04~05 기본):**
- 같은 SKU 1줄 합본 (단일+혼적박스 모두 등장 시)
- 다중 혼적박스 콤마 표기 (예: `12-7, 12-12`)
- declared_box_count 사용자 지정 지원
- 거래명세서 vs S-QED 차이 자동 보정 (ADJUSTMENTS)

## 입력 (사용자 제공)

1. **물류동봉문서 PDF** (쿠팡 거래명세서)
2. **CN인사이더 출고내역 엑셀** (S-QED 형식)

## 작업 흐름

### 1단계: 거래명세서 파싱

PDF에서 추출:
- 업체명, 업체번호
- 발주번호 (요청 ID)
- 운송타입, 물류센터(센터코드 포함, 예: `인천28(INC28)`)
- 도착예정일 (`YYYY-MM-DD` 변환, 원본은 `YYYYMMDD`)
- 팔레트수량 (총 파레트 수)
- 상품 리스트: 상품번호(SKU ID 8자리), 상품명/옵션, 바코드, 확정수량

### 2단계: S-QED 파싱

엑셀 헤더(row 2) 이름으로 열 위치 자동 감지:
- `트레이 번호`, `상자 번호`, `SKU`, `라벨명`, `출고수량`, `세트수량`
- ⚠️ 일부 파일은 `이미지` 추가 컬럼 포함 → 헤더 이름 기반 인덱싱으로 자동 대응
- **세트수량 있으면 우선**, 없으면 출고수량

트레이 prefix 변형:
- `S-QED-G-TP-XXX`: 깔창류 (G = 그로스 일반?)
- `S-QED-B-TP-XXX`: 아쿠아/운동화류 (B = 박스?)
- `S-QED-N-G-TP-XXX`: 가습기류 (N-G = 노트북/뉴트리정 G?)
- 박스 코드도 동일 prefix: `S-QED-G-001`, `S-QED-B-067`, `S-QED-N-G-072` 등

**혼적박스 자동 감지:**
- 같은 (tray, box_code) 조합이 여러 행에 걸쳐 있으면 = 혼적
- 키 = `(tray, box_code)` 단위. 같은 트레이 안에서만.
- → 키-셋 자동 추출 + 사용자 명시 `true_mixed` 합집합 사용

### 3단계: 트레이→파레트 매핑

기본 규칙:
1. G-tray (S-QED-G-TP-XXX) 먼저, 트레이 번호 오름차순
2. N-G-tray (S-QED-N-G-TP-XXX), 트레이 번호 오름차순
3. B-tray (S-QED-B-TP-XXX), 트레이 번호 오름차순

예시 (9파레트):
```
9-1: G-TP-008    9-6: N-G-TP-005
9-2: G-TP-009    9-7: N-G-TP-006
9-3: N-G-TP-002  9-8: B-TP-247
9-4: N-G-TP-003  9-9: B-TP-248
9-5: N-G-TP-004
```

**사용자가 다른 순서를 지정하면 그대로 따른다.**

CN인사이더에서 트레이 정보 없는 경우:
- 사용자에게 트레이 매핑 별도 요청
- 또는 동일 발주의 트레이 정보 있는 파일 추가 요청

### 4단계: 수량 검증 (필수)

거래명세서 vs S-QED 실측 비교:
- SKU별 총수량 비교
- 차이 발견 시 사용자에게 보고 + 단일박스 행에 +차이 보정 (`ADJUSTMENTS` 적용)
- 보정 후 총합 = 거래명세서 총합과 100% 일치해야 함

**전형적 보정 케이스:**
- 깔창/아쿠아 박스 캐파(70~85개)와 발주량 불일치 (±1~2개 흔함)
- 풀컨테이너 분할 입고 시 일부 SKU가 다른 입고에 분배됨

### 5단계: 파레트별 행 생성 (합본 방식)

```
1. 파레트 내 모든 박스(단일+혼적)의 SKU별 수량 합산 → totals[bc]
2. 혼적박스에서 등장한 bc는 mixed_marks[bc]에 박스번호(들) 누적
   - 박스번호 = "{phys_count}-{idx}" (idx는 파레트 내 박스 순번, 1-based)
   - phys_count는 사용자가 declared_box_count를 지정한 경우 그 값 사용
3. 파레트별 총합에 ADJUSTMENTS 적용
4. SKU ID 오름차순 정렬
5. 행 출력: (sku_id, name, box_no, qty)
   - box_no는 mixed_marks[bc] dedup 후 콤마 join (예: "12-7, 12-12"), 없으면 공란
```

**박스 카운트 (총 박스 필드):**
- 실측 = `len(pallet_boxes[p])` (혼적박스도 1개로 카운트)
- 사용자 지정 `declared_box_count` 우선

### 6단계: XLSX + PDF 생성

각 파레트 1페이지(PDF)/1시트(XLSX). 양식:

```
쿠팡 팔레트 적재리스트 (각 팔레트 부착 필수)
※ 팔레트의 높이는 1,700mm를 초과할 수 없습니다 ※

1) 업체 정보
  업체명, 업체코드

2) 입고 예약 정보
  요청 ID, 물류센터, 도착예정일, 팔레트 번호 (bold), 총 박스

3) 상품 정보
  No. | SKU ID | 물류입고용 상품명/옵션명 | 박스 번호 | 상품 수량 | 소비기한/제조일자
```

**PDF 행 높이 자동 조정:**
- 행 수 ≤ 12: 9mm
- 행 수 13~16: 8.5mm
- 행 수 17~20: 7.5mm
- 20행 초과: 7mm (예외적 큰 표)

XLSX는 폰트 "맑은 고딕" 사용. PDF는 NanumGothic (Regular/Bold).

### 7단계: 마운트 폴더 저장

저장 경로:
```
{mount_folder}/적재리스트_{order_id}_{YYYYMMDD_HHMMSS}.xlsx
{mount_folder}/적재리스트_{order_id}_{YYYYMMDD_HHMMSS}.pdf
```

이전에 같은 주문번호 파일이 있어도 timestamp suffix로 새 파일 생성.

## 박스번호 규칙

쿠팡 공식 규칙:
> 박스에 여러 판매 단위 상품을 적재한 경우(혼적박스)에만 `(전체박스수-하위번호)` 작성
> 단일 SKU 박스는 공란
> 각 혼적박스 외부에 동일한 박스 번호 라벨 표시 필수

**용어:**
- **전체박스수** = 해당 파레트의 박스 수 (declared 우선, 없으면 실측)
- **하위번호** = 파레트 내 박스 순번 (1부터, 혼적/단일 무관 카운트)

**사용자 통화 확인 결과:** "파레트별로 작성해주시는게 맞아요"
→ 박스번호는 해당 파레트 기준 (전체 발주 합계 X)

## 엣지 케이스

### B-004 케이스 (같은 라벨, 다른 물리박스)
같은 박스코드가 다른 트레이에 있을 때 = 별개 물리박스 (라벨만 동일).
사용자 명시: "번호 마킹은 동일한데 혼적 돼서 출고가 된게 아니래"
→ 혼적 화이트리스트에서 자동 제외 (auto-detect는 같은 트레이 내에서만 검사).

### G-079 케이스 (실제 혼적, 같은 SKU 다중 행)
G-079에 그린 2 + 블랙 8 + 블랙 6 → 같은 SKU(블랙) 2행은 합산 (블랙 14).
출력: 그린 2 + 블랙 14, 박스번호 모두 동일 (16-15).

### TP-255 부분 파레트
12박스 부분 파레트지만 사용자가 "14박스로 등록" 지정:
- `declared_box_count = {2: 14}` → 박스번호도 14-x 기준
- 총 박스 필드 = 14, 박스번호 = 14-7, 14-12 등

### 혼합 트레이 (G + N-G 혼재)
TP-006이 G-033~G-039 (깔창) + N-G-065~N-G-073 (가습기) 혼합:
- 트레이 prefix 무관하게 같은 트레이의 박스는 같은 파레트로
- 혼적박스 자동 감지는 (tray, box_code) 키 기준이라 정상 작동

### "이미지" 추가 컬럼 (선글라스 케이스)
일부 파일은 헤더에 `이미지` 컬럼 추가 → SKU 열이 4번째로 밀림.
- 헤더 이름 기반 인덱싱으로 자동 대응 (열 번호 하드코딩 X)

### 풀컨테이너 분할 입고
1개 컨테이너를 여러 발주로 분할 시:
- 거래명세서 SKU 합계 ≠ S-QED 합계 가능
- 차이만큼 ADJUSTMENTS로 단일박스 row에 +반영 → 거래명세서 일치
- 사용자에게 차이 발견 시 보고 필수

### 액세서리 포함 셀링 유닛 (선글라스+케이스)
선글라스 SKU는 `8807801050XXX` (색깔별), 케이스는 별도 SKU `CN297120T8450604`로 등장하지만 셀링 단위는 sunglass+case 1세트.
- 케이스는 거래명세서에 별도 SKU로 안 들어감 → S-QED 합계에서 자동 제외
- 박스 안에 sunglass + 케이스 같이 있어도 셀링 SKU만 카운트
- 이 경우는 수동 매핑이 안전 (스크립트는 모든 SKU를 합산하므로 매핑 안 된 SKU는 결과에 안 나타남)

## 사용자 검증 절차

PDF/XLSX 생성 후 반드시:
1. 거래명세서 총합과 일치 여부 출력 (예: `Grand total: 5650 ✓ (거래명세서 5650)`)
2. 혼적박스 목록과 각 박스 안 SKU/수량 명세 출력
3. 박스 외부 마킹은 원래 S-QED 박스코드(B-067, G-079 등) + 적재리스트 박스번호(16-3, 16-15) 같이 부착

`scripts/build_loading_list.py`를 참고. 핵심:

```python
import openpyxl

# 헤더 이름 기반 열 인덱스 감지 (이미지 컬럼 변형 자동 대응)
headers = [str(ws.cell(2,c).value or "") for c in range(1, ws.max_column+1)]
col_tray = headers.index("트레이 번호") + 1

col_sku = headers.index("SKU") + 1
col_ship = headers.index("출고수량") + 1
col_set = headers.index("세트수량") + 1

# 자동 혼적박스 감지 (같은 트레이 내 같은 박스코드 다중 행)
# + 사용자 명시 화이트리스트 합집합

# 합본 행 생성 (mixed_marks dedup + 콤마 join)
# ADJUSTMENTS 적용

# PDF 행 높이 자동 결정
def row_h(n):
    if n <= 12: return 9
    if n <= 16: return 8.5
    if n <= 20: return 7.5
    return 7
```

## 회송분(반품 재작업) 적재리스트 처리 (v1.3.0+)

표준 입고분과 **입력 양식이 다른** 회송(반품) 건은 별도 스크립트
`scripts/build_return_loading_list.py`로 처리한다.

### 회송 양식 식별
- 엑셀이 **멀티시트**이고 시트명이 `10-01`, `10-02` … 처럼 파레트 단위 (시트 1개 = 파레트 1개).
- 각 시트 헤더(보통 3행): `바코드끝6자리`, `상품명`, `옵션`, `입수`, `박스수량`, `총수량`.
- **박스수량 컬럼의 병합셀 = 박스 1개** (병합된 여러 행 = 같은 박스에 든 혼합 품목).
- 상품코드는 6자리뿐이므로 **동봉문서 바코드 끝 6자리**로 SKU·상품명을 매칭한다.

### 작업 흐름
1. 동봉문서(거래명세서) PDF를 파싱해 `bc6_to_product = { 바코드끝6자리: [SKU, 상품명/옵션, 동봉No] }` 구성.
2. config(JSON) 작성:
   ```json
   {
     "order_id": "...", "company_name": "...", "company_code": "...",
     "center": "...", "arrival_date": "YYYY-MM-DD",
     "src_xlsx": "/회송엑셀.xlsx",
     "bc6_to_product": { "050254": ["59835158", "이더커머스 ... 그레이 235-240", 1], ... },
     "code_fix": { "055156": "050156" },        // (선택) 6자리 오타 보정
     "delivery_total": { "SKU": qty, ... }       // (선택) 동봉 발주합 초과검증용
   }
   ```
3. 실행:
   ```
   python scripts/build_return_loading_list.py --config <config.json> --output-dir <폴더>
   ```
4. 출력: SKU/옵션당 1줄 합본 + 혼적박스만 콤마 박스번호(`{박스수}-{순번}`), 파레트당 1페이지.

### 검증 (회송도 동일 원칙)
- **초과(적재 수량 > 동봉 발주수량) 0건**이어야 안전. 부족분은 정상(분할 선적).
- 박스수·SKU별 수량·박스번호를 엑셀에서 독립 재계산해 대조 권장.
