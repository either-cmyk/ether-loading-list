# 쿠팡 팔레트 적재리스트 생성 스킬 (v2.0 합본 방식)

## 개요

거래명세서 PDF + CN인사이더 출고내역 엑셀(S-QED) 두 파일을 받아 **팔레트별 1페이지**로 구성된 적재리스트 XLSX + PDF를 동시 생성한다.

**v2.0 핵심 변경점 (2026-04~05 사용자 피드백 반영):**
- 같은 SKU가 단일박스 + 혼적박스에 모두 있으면 **1줄로 합본**, 박스번호만 표기 (이전 v1: SKU별 분리 다중행)
- 한 SKU가 여러 혼적박스에 걸치면 **콤마 다중 표기** (예: `12-7, 12-12`)
- 박스수 사용자 지정 가능 (실제 박스 < 등록 박스 시 declared 우선, 박스번호도 declared 기준)
- 거래명세서 vs S-QED 수량 차이 자동 보정 (단일박스 row에 +adjustment)

## 입력 (사용자 제공)

1. **물류동봉문서 PDF** (쿠팡 거래명세서)
2. **CN인사이더 출고내역 엑셀** (S-QED 형식, FBC창고 → 입고제품 → 해당 건 상세의 "Excel 다운로드")

## 작업 흐름

### 1단계: 거래명세서 파싱

PDF에서 추출:
- 업체명, 업체번호
- 발주번호 (요청 ID)
- 운송타입, 물류센터(센터코드 포함, 예: `인천28(INC28)`)
- 도착예정일 (`YYYY-MM-DD` 변환)
- 팔레트수량 (총 파레트 수)
- 상품 리스트: 상품번호(SKU ID 8자리), 상품명/옵션, 바코드, 확정수량

### 2단계: S-QED 파싱

엑셀에서 트레이별 박스 추출:
- 트레이 번호 (예: `S-QED-G-TP-008`, `S-QED-B-TP-254`)
- 상자 번호 (예: `S-QED-G-001`, `S-QED-B-067`)
- SKU 바코드, 라벨명
- 출고수량(K열)과 세트수량(L열) — **세트수량 있으면 우선**, 없으면 출고수량

**혼적박스 판별 (중요):**
- 같은 (트레이, 상자번호) 조합이 여러 행에 걸쳐 있으면 = 1개 물리박스 안에 다중 SKU = **혼적박스**
- 단, **B-004 케이스 주의**: 같은 박스코드라도 다른 트레이에 있으면 별도 물리박스 (라벨만 같음, 혼적 아님)
- → 혼적박스 키 = `(tray, box_code)` 단위로 판별. 같은 트레이 안에서만.

### 3단계: 트레이→파레트 매핑

기본 규칙:
- G-tray 먼저, 트레이 번호 오름차순
- 그 다음 B-tray, 트레이 번호 오름차순

예시 (13파레트):
```
13-1: G-TP-008    13-6: B-TP-248
13-2: G-TP-009    13-7: B-TP-249
13-3: G-TP-010    13-8: B-TP-250
13-4: G-TP-011    13-9: B-TP-251
13-5: G-TP-012    13-10: B-TP-252
                  13-11: B-TP-253
                  13-12: B-TP-254
                  13-13: B-TP-255
```

**사용자가 다른 순서를 지정하면 그대로 따른다.** (예: "008번이 1번이야")

CN인사이더에서 트레이 정보 없는 경우:
- 사용자에게 트레이→파레트 매핑 별도 요청
- 또는 동일 발주번호의 트레이 정보 있는 파일(S-QED-G_xxx.xlsx) 추가 요청

### 4단계: 수량 검증 (필수)

거래명세서 vs S-QED 실측 비교:
- SKU별 총수량 비교
- 차이 발견 시 사용자에게 보고 + 단일박스 행에 +차이만큼 보정 (`ADJUSTMENTS` 적용)
- 보정 후 총합 = 거래명세서 총합과 100% 일치해야 함

**보정이 필요한 전형적 케이스:**
- 깔창 박스 캐파(70~85개)와 발주량 불일치
- 풀컨테이너를 여러 입고건으로 분할할 때 일부 SKU가 다른 입고에 분배됨

### 5단계: 파레트별 행 생성 (합본 방식)

**합본 로직 (v2.0 핵심):**
```
1. 파레트 내 모든 박스(단일+혼적)의 SKU별 수량 합산 → totals[bc]
2. 혼적박스에서 등장한 bc는 mixed_marks[bc]에 박스번호(들) 누적
   - 박스번호 = "{phys_count}-{idx}" (idx는 파레트 내 박스 순번, 1-based)
   - phys_count는 사용자가 declared_box_count를 지정한 경우 그 값 사용
3. 파레트별 총합에 ADJUSTMENTS 적용
4. SKU ID 오름차순 정렬
5. 행 출력: (sku_id, name, box_no, qty)
   - box_no는 mixed_marks[bc] dedup 후 콤마 join (예: "12-7, 12-12"), 없으면 공란
```

**박스 카운트 (총 박스 필드):**
- 실측 = `len(pallet_boxes[p])` (혼적박스도 1개로 카운트, 다중 행 ≠ 다중 박스)
- 사용자 지정이 있으면 declared 우선

### 6단계: XLSX + PDF 생성

각 파레트 1페이지(PDF)/1시트(XLSX), 양식:

```
쿠팡 팔레트 적재리스트 (각 팔레트 부착 필수)
※ 팔레트의 높이는 1,700mm를 초과할 수 없습니다 ※

1) 업체 정보
  업체명: {company_name}
  업체코드: {company_code}

2) 입고 예약 정보
  요청 ID: {order_id}
  물류센터: {center}
  물류센터 도착예정일: {arrival_date}
  팔레트 번호: {total_pallets}-{pallet_no}   ← bold
  총 박스: {phys_count}

3) 상품 정보
  No. | SKU ID | 물류입고용 상품명 / 옵션명 | 박스 번호 | 상품 수량 | 소비기한/제조일자
  1   | ...    | ...                        | (공란/16-15) | ... | -
```

폰트: NanumGothic (Regular/Bold). PDF는 reportlab, XLSX는 openpyxl.

### 7단계: 마운트 폴더 저장

저장 경로 (사용자가 폴더 mount했다고 가정):
```
{mount_folder}/적재리스트_{order_id}_{YYYYMMDD_HHMMSS}.xlsx
{mount_folder}/적재리스트_{order_id}_{YYYYMMDD_HHMMSS}.pdf
```

이전에 같은 주문번호 파일이 있어도 timestamp suffix로 새 파일 생성 (덮어쓰기 X).

## 박스번호 규칙 (쿠팡 공식 + 사용자 검증)

쿠팡 공지 그대로:
> 박스에 여러 판매 단위 상품을 적재한 경우(혼적박스)에만 `(전체박스수-하위번호)` 작성
> 단일 SKU 박스는 공란
> 각 혼적박스 외부에도 동일한 박스 번호 라벨 표시 필수

**용어 통일:**
- **전체박스수** = 해당 파레트의 박스 수 (declared 우선, 없으면 실측)
- **하위번호** = 파레트 내 박스 순번 (1부터, 혼적이든 단일이든 모든 박스 포함하여 카운트)

**사용자 통화 확인 결과 (2026-04 콜):** "파레트별로 작성해주시는게 맞아요"
→ 13-2 파레트의 16번째 박스가 혼적이면 `16-16` (전체 39 기준 아님)

## 합본 방식 예시

**Pallet 13-13 (12박스, B-119/B-124 혼적):**

| No | SKU ID | 옵션 | 박스 번호 | 수량 |
|---|---|---|---|---|
| 1 | 44337014 | 블랙 250-255 |  | 150 |
| 2 | 44337018 | 블랙 240-245 |  | 70 |
| 3 | 44337023 | 블루 220-225 | **12-7, 12-12** | 60 |
| 4 | 44337026 | 피치 220-225 |  | 150 |
| 5 | 44337029 | 퍼플 210-215 | **12-7** | 60 |
| 6 | 44337039 | 블랙 220-225 | **12-12** | 100 |

블루 220-225는 B-119(20개) + B-124(40개) = 60개를 1줄로 합산 + 박스번호 두 개 표기.

## 엣지 케이스

### B-004 케이스 (같은 라벨, 다른 물리박스)
TP-246 안에 B-004가 두 행(빗자루 25 + 수세미 250)으로 있을 때:
- 두 트레이는 동일하지만 **물리적으로 분리된 2박스** (마킹만 동일)
- 사용자 명시: "번호 마킹은 동일한데 혼적 돼서 출고가 된게 아니래"
- → 혼적이 아닌 별개 박스 2개로 처리. `TRUE_MIXED` 화이트리스트에서 제외.

### G-079 케이스 (같은 박스, 같은 SKU 다중 행)
```
G-079 / 그린, 2
G-079 / 블랙, 8
G-079 / 블랙, 6
```
같은 SKU 2행은 합산 (블랙 14), → 그린 2 + 블랙 14로 표기, 박스번호 모두 동일 (예: 16-15).

### TP-255 같은 부분파레트
- 12박스 부분파레트지만 사용자가 "16박스로 등록" 지정 → declared=16 사용, 박스번호도 16-x

### 트레이 정보 누락
S-QED 파일에 트레이 컬럼이 비어있으면:
1. 사용자에게 트레이 정보 있는 동일 발주의 다른 파일 요청
2. 또는 박스 수동 분배 안내

### 풀컨테이너 분할 입고
1개 컨테이너를 여러 발주로 분할 입고 시:
- 거래명세서 SKU 합계 ≠ S-QED 합계 가능
- 차이만큼 단일박스 row에 +adjustment 적용 → 거래명세서 일치시킴

## 사용자 검증 절차

PDF/XLSX 생성 후 반드시:
1. 거래명세서 총합과 일치 여부 출력 (예: `Grand total: 4430 (거래명세서 4430)`)
2. 혼적박스 목록과 각 박스 안 SKU/수량 명세 출력
3. 박스 외부 마킹은 원래 S-QED 박스코드(B-067, G-079 등) 그대로 두고, 적재리스트의 박스번호(16-3, 16-15)도 같이 부착하면 쿠팡 입고팀이 매칭 가능함을 안내

## 실행 코드 패턴 (Python)

scripts/build_loading_list.py를 참고. 핵심 흐름:

```python
import openpyxl
from collections import OrderedDict

# 1. 매핑 정의
TRAY_TO_PALLET = {...}
TRUE_MIXED = {...}  # 혼적박스 화이트리스트 (트레이별 같은 박스코드 다중 행 자동감지로 채움)
BC_TO_PRODUCT = {...}  # 바코드 → (SKU_ID, 상품명/옵션)
ADJUSTMENTS = {...}  # (pallet, bc) -> +delta
DECLARED_BOX_COUNT = {...}  # {pallet_no: declared_count} - 사용자 지정 시

# 2. 파싱: pallet_boxes[p] = [(box_code, OrderedDict(bc->qty)), ...]

# 3. 합본 행 생성
def build_rows(p):
    boxes = pallet_boxes[p]
    phys = len(boxes)
    declared = DECLARED_BOX_COUNT.get(p, phys)
    totals = OrderedDict()
    mixed_marks = {}
    for idx, (bc_box, skus) in enumerate(boxes, 1):
        is_mixed = bc_box in TRUE_MIXED
        for bc, q in skus.items():
            totals[bc] = totals.get(bc, 0) + q
            if is_mixed:
                mixed_marks.setdefault(bc, []).append(f"{declared}-{idx}")
    for (pp, bc), delta in ADJUSTMENTS.items():
        if pp == p and bc in totals:
            totals[bc] += delta
    rows = []
    for bc in sorted(totals.keys(), key=lambda x: int(BC_TO_PRODUCT[x][0])):
        marks = mixed_marks.get(bc, [])
        seen = set(); uniq = []
        for m in marks:
            if m not in seen: seen.add(m); uniq.append(m)
        sku, name = BC_TO_PRODUCT.get(bc, ("", bc))
        rows.append((sku, name, ", ".join(uniq) if uniq else "", totals[bc]))
    return declared, rows
```
