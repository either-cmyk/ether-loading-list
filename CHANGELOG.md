# 변경 이력

## v1.3.0 - 2026-06-24

### 회송분(반품 재작업) 적재리스트 양식 지원 (신규)

- **회송 양식 자동 처리**: 멀티시트 엑셀(시트 1개 = 파레트, 예 `10-01`=1번 파레트) +
  박스수량 컬럼 **병합셀 = 박스 1개** 구조 인식. 병합 그룹 = 같은 박스 혼합 품목.
- **바코드끝6자리 매칭**: 회송 엑셀의 6자리 코드를 동봉문서 바코드 끝 6자리로 SKU 매핑.
- **바코드 오타 보정(code_fix)**: 6자리 코드 오타를 수량 정합 기반으로 보정 (선택).
- **회송 전용 PDF 렌더**: 박스번호 콤마 리스트가 길어도 겹치지 않게 박스 컬럼 확대 +
  행높이 자동(가변). 표준 입고분 렌더는 그대로 유지.
- **신규 스크립트**: `scripts/build_return_loading_list.py` (PDF 전용, XLSX는 공용 재사용)
- **PDF 행높이 단계 확장**: 21~44행 이상도 1페이지 fit (조밀 단계 추가)
- **plugin.json 정상화**: keywords 배열이 닫히지 않은 깨진 JSON 수정 + `return` 키워드 추가
- **검증**: 134329099 (10파레트 258박스, 2906개) — 엑셀 독립 재계산 대비 박스수·수량·박스번호
  불일치 0건, 동봉 발주합 2909 대비 초과 0건.


## v1.2.0 - 2026-06-24

### 신규 스킬: cn-incoming-converter (에이투지 간편입고 등록 변환)

- CN인사이더 출고내역(S-QED / S-QDE) → 사방넷 풀필먼트 **간편입고 등록** 엑셀 자동 변환
- **상품코드표 내장** (`product_codes.json`, 라벨명→E코드 597건) → 별도 공산품 상품코드 파일 업로드 불필요
- 라벨명(H열) 정규화 매칭(공백/끝콤마 제거)으로 상품코드 자동 부여, 바코드(E열)는 출고내역 SKU 직접 사용
- 수량 = 세트수량(L열) 우선, 없으면 출고수량(K열) / 바코드 기준 합산 / 라벨명 오름차순 정렬
- 박스번호 연속 범위(`~`)·불연속(`,`) 표기, S-QED·S-QDE 코드 모두 인식
- 내부 합계 검증(그룹합 = 행합)으로 누락 방지
- 검증된 출고내역: S-QED260529(글램루아 브라 5종 100세트), S-QDE260529(클린인테크 양말 5종 17,000세트/680박스), S-QED260618(깔창 6종 1,049세트, 혼적박스 포함)

## v1.1.0 - 2026-05-18

### 트레이 다종 지원 + 헤더 기반 인덱싱 + PDF 행 높이 자동 조정

- **N-G 트레이 지원**: `S-QED-N-G-TP-XXX` (가습기류) prefix 추가 인식
- **혼합 트레이 지원**: 한 트레이에 G/N-G/B 다른 prefix 박스가 섞여 있어도 정상 처리
- **헤더 이름 기반 컬럼 인덱싱**: `이미지` 추가 컬럼 변형 자동 대응 (선글라스 케이스 등)
- **PDF 행 높이 자동 조정**:
  - ≤12행: 9mm / 13-16행: 8.5mm / 17-20행: 7.5mm / 20행 초과: 7mm
  - 18 SKU 이상 발주 (선글라스 다색상)도 A4 1페이지 fit
- **트레이 매핑 우선순위 명시**: G → N-G → B, 각 그룹 내 트레이번호 오름차순
- **액세서리 SKU 자동 제외**: 매핑 안 된 SKU (예: 케이스 `CN297120T8450604`)는 결과에서 자동 제외
- **검증된 신규 케이스**:
  - 131385250 (1파레트 11박스) — 선글라스 + 우양산 18 SKU, "이미지" 컬럼 변형
  - 131638295 (9파레트 140박스) — G/N-G/B 트레이 혼합, 8개 혼적박스
