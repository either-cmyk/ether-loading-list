같은 SKU(같은 옵션)이 단일박스 + 혼적박스에 모두 있어도 **1줄로 합본**.
박스번호는 혼적박스에 들어간 부분만 표기.

**Why:** 사용자가 "사이즈가 동일한데 굳이 분리할 필요 없잖아 / 그냥 하나에 박스번호만 쓰면 되는 거 아니야?" 라고 명확히 요청.

**How to apply:**
- SKU별 totals[bc]에 단일+혼적 모두 합산
- 혼적박스에서 등장한 SKU만 mixed_marks[bc]에 박스번호 누적
- 출력 시 1줄, 박스번호는 dedup 후 콤마 join (예: `12-7, 12-12`)
- 단일박스만의 SKU는 박스번호 공란

**예시 (Pallet 13-13 / 12박스):**

| SKU | 옵션 | 박스번호 | 수량 |
|---|---|---|---|
| 44337014 | 블랙 250-255 | (공란) | 150 |
| 44337023 | 블루 220-225 | **12-7, 12-12** | 60 (B-119 20 + B-124 40) |
| 44337029 | 퍼플 210-215 | **12-7** | 60 |
| 44337039 | 블랙 220-225 | **12-12** | 100 (단일 70+2 + B-124 28) |

블루 220-225는 두 혼적박스에 걸쳐있어서 박스번호 두 개 콤마 표기.

**박스 외부 라벨**: 원래 S-QED 박스코드(B-119, B-124) 그대로 유지 + 적재리스트 박스번호(12-7, 12-12)도 같이 부착. 쿠팡 입고팀이 매칭 가능.

**이전 v1 방식과 차이:**
- v1: 같은 SKU지만 단일/혼적 분리해서 2~3줄 출력 (혼적 줄에 박스번호)
- v2: 1줄로 합본, 박스번호 콤마 표기

이전 방식으로 돌리고 싶으면 mixed_rows를 별도로 두고 출력 시 분리 (build_rows 함수 수정).
