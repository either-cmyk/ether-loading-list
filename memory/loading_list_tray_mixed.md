## 트레이 prefix 종류 (CN인사이더)

S-QED 출고내역 파일의 트레이/박스 코드 prefix:

| Prefix | 의미 | 예시 |
|---|---|---|
| `S-QED-G-TP-XXX` | G(그로스) 일반 - 깔창류 | `S-QED-G-TP-008`, `S-QED-G-014` |
| `S-QED-B-TP-XXX` | B(박스) - 아쿠아/운동화/선글라스 | `S-QED-B-TP-247`, `S-QED-B-067` |
| `S-QED-N-G-TP-XXX` | N-G - 가습기류 | `S-QED-N-G-TP-006`, `S-QED-N-G-072` |

## 트레이→파레트 매핑 기본 규칙

순서: **G → N-G → B**, 각 그룹 내에서 트레이 번호 오름차순.

예시 (9파레트):
```
9-1: G-TP-008    9-6: N-G-TP-005
9-2: G-TP-009    9-7: N-G-TP-006
9-3: N-G-TP-002  9-8: B-TP-247
9-4: N-G-TP-003  9-9: B-TP-248
9-5: N-G-TP-004
```

**사용자가 다른 순서 지정 시 그대로 따름** (예: "008번이 1번이야")

## 혼합 트레이 케이스

한 트레이에 다른 prefix 박스가 같이 들어가는 경우:
- `S-QED-N-G-TP-006` 안에 `S-QED-G-033~G-039` (깔창) + `S-QED-N-G-065~N-G-073` (가습기)
- 같은 트레이의 박스는 모두 같은 파레트
- 혼적박스 자동 감지는 `(tray, box_code)` 키 기준이라 정상 작동

## 혼적박스 자동 감지

**자동 감지 로직:**
- 같은 (트레이, 박스코드) 조합이 여러 행에 걸쳐 있으면 혼적
- `box_skus[(tray, box_code)] = set(sku_list)` → len > 1 이면 혼적
- 사용자 명시 `true_mixed` 리스트와 합집합

## 주의 케이스

### B-004 케이스 (라벨만 같음, 별개 박스)
같은 박스코드라도 다른 트레이에 있으면 별개 물리박스 (마킹 우연히 동일).
사용자 명시: "번호 마킹은 동일한데 혼적 돼서 출고가 된게 아니래"
→ 자동 감지는 같은 트레이 내에서만 검사하므로 정상 처리됨.

### G-079 케이스 (실제 혼적, 같은 SKU 다중 행)
TP-012 안에 G-079가 3행 (그린 2, 블랙 8, 블랙 6).
- 같은 박스 안 그린 + 블랙 혼적 (실제 혼적)
- 블랙은 행 분리됐지만 같은 SKU → 합산 (블랙 14)
- 출력: 그린 2 + 블랙 14, 박스번호 동일 (예: 16-15)

### B-119/B-124 다중 혼적 케이스
- B-119: 퍼플 60 + 블루 20 (mixed)
- B-124: 블루 40 + 블랙 28 (mixed)
- 블루 SKU는 양쪽에 등장 → "12-7, 12-12" 콤마 표기

### 풀컨테이너 분할 입고
1개 컨테이너(예: 79박스)를 여러 쿠팡 발주로 분할 시:
- 거래명세서 수량 ≠ S-QED 합계 가능
- ADJUSTMENTS로 단일박스 row에 +반영해서 거래명세서 일치
- 사용자에게 차이 발견 시 보고 필수

### 트레이 정보 누락
S-QED 파일에 트레이 컬럼 비어있을 시:
- 사용자에게 트레이 정보 있는 동일 발주의 다른 파일 요청
- 또는 박스 분배 수동 안내

### "이미지" 추가 컬럼 (선글라스 케이스)
일부 파일은 헤더에 `이미지` 컬럼 추가 → SKU 열이 4번째로 밀림.
**해결:** 헤더 이름 기반 인덱싱 (열 번호 하드코딩 X). 빌더 v1.1.0 자동 대응.

### 액세서리 SKU (선글라스 케이스)
선글라스 SKU는 `8807801050XXX` (색깔별), 케이스는 `CN297120T8450604`.
- 케이스는 거래명세서에 별도 SKU로 안 들어감
- 셀링 유닛은 sunglass+case 1세트
- 매핑 안 된 SKU는 결과에 자동 제외됨

## 부분 파레트 (declared_box_count)

실측 박스가 12개지만 사용자가 14박스로 등록한 경우:
```json
"declared_box_count": {"2": 14}
```
- 총 박스 필드 = 14
- 박스번호도 14-x 기준 (12-x 아님)
