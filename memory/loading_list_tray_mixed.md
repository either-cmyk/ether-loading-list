## 트레이→파레트 매핑

기본 규칙:
1. G-tray (S-QED-G-TP-XXX) 먼저, 트레이 번호 오름차순
2. 그 다음 B-tray (S-QED-B-TP-XXX), 트레이 번호 오름차순

**예시 (CN인사이더 13파레트 발주):**
- 13-1: G-TP-008  /  13-2: G-TP-009  /  13-3: G-TP-010  /  13-4: G-TP-011  /  13-5: G-TP-012
- 13-6: B-TP-248  /  13-7: B-TP-249  /  ...  /  13-13: B-TP-255

**사용자가 다른 순서 지정 시 그대로 따름** (예: "008번이 1번이야")

## 혼적박스 판별

**혼적박스 정의:** 1개 물리 박스 안에 여러 SKU가 들어있는 박스.

**자동 감지 로직:**
- 같은 (트레이, 박스코드) 조합이 여러 행에 걸쳐 있으면 혼적
- 키 = `(tray, box_code)`. 트레이가 다르면 별개 박스로 처리.

## 주의 케이스

### B-004 케이스 (라벨만 같음, 별개 박스)
이전 발주(TP-246)에서 B-004가 두 행 (빗자루 25, 수세미 250)으로 등장.
사용자 명시: **"번호 마킹은 동일한데 혼적 돼서 출고가 된게 아니래. 빗자루랑 수세미는 분리해주면 될듯?"**
→ 같은 박스코드라도 다른 트레이에 있으면 별개 물리박스 (마킹 우연히 동일).
→ 혼적 화이트리스트에서 제외.

### G-079 케이스 (실제 혼적, 같은 SKU 다중 행)
TP-012 안에 G-079가 3행 (그린 2, 블랙 8, 블랙 6).
- 같은 박스 안에 그린 + 블랙 혼적 (실제 혼적)
- 단, 블랙은 행 분리됐지만 같은 SKU → 합산해서 블랙 14로 1줄
- 출력: 그린 2 + 블랙 14, 박스번호 동일 (예: 16-15)

### 풀컨테이너 분할 입고
1개 컨테이너(예: 79박스)를 여러 쿠팡 발주로 분할 입고하는 경우:
- 거래명세서 수량 ≠ S-QED 합계 가능
- 차이만큼 ADJUSTMENTS로 단일박스 row에 +반영해서 거래명세서 일치시킴
- 사용자에게 차이 발견 시 보고 필수

### 트레이 정보 누락
S-QED 파일에 트레이 컬럼 비어있을 시:
- 사용자에게 트레이 정보 있는 동일 발주의 다른 파일 요청
- 또는 박스 분배 수동 안내 받기
